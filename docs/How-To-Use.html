1. Overview

This is a meeting minutes and action tracking system built in Excel. Take notes during the meeting, mark anything that needs follow-up as Action or Risk, and use the AR view to see all open actions filtered.

Compared to the old version: no checkboxes (use the Type dropdown), no separate Action Register sheet (it's a filter view of Meeting Minutes), no two-way sync to break, and project structure is driven by where rows sit on the sheet.

3. Meeting Minutes structure

Header block (rows 1–14)

Title, location, date, attendees, keyboard shortcuts panel. Don't edit the structure here — just update the date and tick off attendees each meeting.

Column headers (row 15)

ColHeaderWhat it holds
ATypeDropdown: Action / Risk / blank. Blank = general note. Drives the row's CF.
BDescriptionWhat was discussed. One clear point per row.
COwnerDropdown: TM1 / TM2 / TM3 / TM4 / TM5 / TM6 / ALL.
DDueDate the action must be done. DD/MM/YYYY.
EStatusDropdown: Open / In Progress / Done / On Hold / Waiting.
FPriorityDropdown: Critical / High / Medium / Low.
GDays ODAuto-calculated days overdue. Blank if not overdue or not an action.
HNotes / UpdateFree text. Use for weekly progress or context.
IProject (hidden)Project name on every row. Used by AR view filter. Filled by AddRow / AddProject macros.
JDate Added (hidden)Auto-stamped on row creation by AddRow. Never overwritten.

Banners

Two banner levels stacked above each project's data rows:

  • Lifecycle banner (full-width, dark colour): TEAM, ACTIVE PROJECTS, DLP PROJECTS, FUTURE PROJECTS, STRATEGIC, GENERAL BUSINESS, CLOSED PROJECTS.
  • Project banner (full-width, light blue): one per project under its lifecycle. E.g. "Project A".

Lifecycle = where a project sits in the sheet. To move a project from Active to DLP: select the project's banner + all its data rows, cut, paste under the DLP banner. No macro needed.

5. Weekly workflow

Tuesday — during the meeting

  1. Open the file from the synced SharePoint folder. Update the date in the header.
  2. Tick off attendees in the attendees table.
  3. Take minutes row by row. Ctrl + Shift + A adds a new row under the current project. Ctrl + Shift + P adds a new project banner if needed.
  4. For anything requiring follow-up, set Type = Action. Fill in Owner, Due, Status, Priority. Row turns amber automatically.
  5. For anything that's a risk to flag, set Type = Risk.
  6. Save the file.

Tuesday to Monday — during the week

  1. Open the file. Use the Action Register filter button (or run ShowARView macro) to see only your actions.
  2. Update Status as you complete actions. Add notes in col H if helpful.
  3. If new risks come up, add them to the Risk Register tab directly.
  4. Save when done.

Next Tuesday — before the meeting

  1. Open the file. Click Hide / Show Done button to declutter (hides Done rows).
  2. Review the Summary tab — see who has outstanding actions.
  3. Take this week's minutes continuing from where you left off.

7. Troubleshooting

ProblemFix
Row not turning amberCheck Type column (col A) — must be exactly "Action" from the dropdown.
AddRow inserts in the wrong placeClick a data row first (row 16+), then run. It inserts above the active row.
Action Register filter not showing my rowCheck that col I (Project) is filled. AR view filters by Project, so blank Project rows are hidden.
Ctrl+Shift+A doesn't workMacro not bound. Alt+F8 → AddRow → Options → assign Ctrl+Shift+A.
Status / Priority colour wrongOpen Manage Rules (Home → Conditional Formatting → Manage Rules). Make sure Status/Priority cell rules are above the row-level rules in priority order.
File saving to wrong placeAlways open from the synced SharePoint folder in File Explorer, not from the browser.
Macro security warningClick Enable Content in the yellow bar at the top.
Need to dig deeper? Anything not in this guide — check with Adina or open the macro archive HTML for the technical history.
