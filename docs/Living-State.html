2. Locked-in decisions

Architecture (Option 3 — simplified)

  • Project Register: DELETED. Lifecycle determined by row position in MM. Manual cut/paste between sections.
  • Lifecycle values (7): Active, DLP, Future, Closed, Team, Strategic, General.
  • Action Register: NO separate sheet. AR view = ShowActionRegisterView macro filtering MM rows where Type=Action.
  • Banners: manually managed. NO auto-rebuild macro — RebuildBanners deleted as too destructive.
  • Single master file — no weekly file rotation. New minutes added on top.
  • Tab name: "Meeting Minutes" (not "Minutes & Actions").

MM column structure (FINAL)

  • A = Type (dropdown: Action, Risk, blank)
  • B = Description
  • C = Owner (dropdown: TM1, TM2, TM3, TM4, TM5, TM6, ALL — no AL)
  • D = Due (DD/MM/YYYY)
  • E = Status (dropdown: Open, In Progress, Done, On Hold, Waiting)
  • F = Priority (dropdown: Critical, High, Medium, Low)
  • G = Days OD (formula)
  • H = Notes / Update
  • I = Project (hidden, used for AR filter)
  • J = Date Added (hidden, auto-stamped on row creation) — being added now

3. Claude's failure modes

Read this BEFORE acting on screenshots, file uploads, or "fix this" requests. These have all happened in this project.

Misreading screenshots

  • Confused adjacent pink/amber colours — call out the actual hex, don't rely on quick glance.
  • Missed banner positions — verify with openpyxl read, not just image.

Forgetting MM header block

  • Header block is rows 1–14, cols A–J. Includes title/date/attendees/shortcuts panel.
  • NEVER suggest column-delete operations — would destroy header.
  • All data macros start at row 16 minimum (row 15 = column headers).

Generating overlong responses

  • User has ADHD. Short, granular, sequential steps only. No multi-bucket plans before checking actual file state.
  • If a response feels long, split across multiple turns.

Generating plans before reading actual file state

  • Always read the uploaded file with openpyxl FIRST when diagnosing.
  • "It should work" answers without inspection have been wrong multiple times.

Contradicting established project decisions

  • Read this doc before suggesting architectural changes.
  • Project Register deleted, AR is filter view not sheet, no auto-banner rebuild — these are LOCKED.

Forgetting bookmarked items

  • "Bookmarked" alone = item gets dropped within 5–10 turns.
  • Real fix: write into the Pending Tasks section above immediately, then re-export this file.
  • Date Added kept getting dropped through phase reorders before being fixed.

Not picking the user's stated best option

  • If user says "go with the best", do that. Don't fall back to second-best because it seems easier.
  • Specifically: when user said "make it into a html or something I can save to project" — that means create the HTML and present it, not "second best, add to existing doc."

VBA gotchas

  • Reserved word any cannot be used as a variable name.
  • VBA line continuation limit — flatten nested arrays via helper subs.
  • Dropdown syntax: Formula1:="Active,DLP,Closed" — no triple quotes (Error 023).
  • Shape .TextFrame.Characters.Text can throw on shapes without text frames — guard with If shp.HasTextFrame Then.
